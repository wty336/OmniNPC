### 项目名称：OmniNPC - 面向高自由度互动游戏（AVG）的认知架构 Agent 引擎

**一、 项目一句话介绍 (Elevator Pitch)**

本项目针对传统互动文字游戏（如橙光、易次元）中剧情分支穷举困难、NPC 缺乏灵魂的痛点，构建了一套**基于大模型认知架构与 GraphRAG 图谱记忆的可拔插式 NPC 后端引擎**。彻底打破固定选项的限制，实现了玩家自由交互与游戏世界真实动态演化的完美闭环。

**二、 项目核心内容 (Project Overview)**

本项目并非开发一款独立游戏，而是打造一个**“为纸片人注入灵魂”的中间件引擎**。玩家可以通过自然语言与游戏角色自由对话，系统会在后台实时处理玩家的输入，驱动 NPC 进行“感知-反思-决策”。引擎不仅会生成符合人设的动态台词反馈给前端，还会通过大模型的工具调用（Function Calling）实时回调并修改游戏底层的数值状态（如好感度、阵营声望、持有道具）。同时，NPC 之间在后台会自动进行记忆同步与社交演化，构建出一个真正“牵一发而动全身”的活体游戏沙盒。

**三、 核心技术亮点 (Key Highlights)**

- **首创仿生分层记忆引擎 (Bionic Layered Memory System)**
  - 摒弃了传统的全局上下文堆叠（极易导致 Token 爆炸和幻觉），自主设计了**工作记忆（短期）、情景记忆（事件）和语义记忆（客观认知）**的三层架构。
  - 创新性地引入了 **艾宾浩斯遗忘曲线（Ebbinghaus Forgetting Curve）与情绪权重打分**机制。NPC 会自然遗忘日常闲聊，但会永久铭记引发其强烈情绪波动（如背叛、救命之恩）的关键交互，实现了极度拟人化的长程陪伴感。
- **基于 GraphRAG 的复杂社会网络演化 (Dynamic Graph-based RAG)**
  - 从底层的向量检索升级为图数据库驱动。实时从玩家与 NPC 的交互文本中抽取实体（Entity）与关系（Relation），自动构建全局动态知识图谱。
  - 突破了传统单线剧情的局限，NPC 可通过图谱关系进行多级跳跃推理（例如：通过图谱检索发现玩家暗中接触了敌对阵营，从而在对话中生成试探或防备的特殊剧情）。
- **双层思维链与状态回调闭环 (Inner Monologue & State-Driven Callbacks)**
  - 引入了“内心独白层（Inner Monologue）”。强制 Agent 在输出玩家可见的台词前，先结合自身性格与当前好感度生成一段“后台心理活动”，彻底解决 NPC 动机不明和“讨好型人格”（OOC）问题。
  - 结合大模型强大的 Function Calling 能力，实现了对话文本与游戏底层数值的解耦与联动（所谓“言出法随”），保证了高自由度交互下游戏核心逻辑的不崩塌。
- **离线社会涌现机制 (Offline Social Emergence)**
  - 设计了基于后台 Tick 的异步多智能体推演流。即使玩家不与某位 NPC 交互，NPC 之间也会自主进行“流言传播”与“状态传染”（如 NPC A 将玩家的秘密告诉 NPC B），让玩家感受到游戏世界在独立于自己真实运转。

**四、 预期技术栈 (Tech Stack)**

- **大模型调度库：** LangChain / LlamaIndex（用于构建工作流和路由）
- **记忆与数据库：** NetworkX / Neo4j（图数据库，处理关系图谱）、ChromaDB（向量数据库，处理文本检索）
- **核心服务：** FastAPI / Python（构建高性能后端 API，便于随时接入任何前端/游戏引擎）
- **展示前端：** Streamlit / Gradio（用于快速验证逻辑和构建面试演示 Demo）

### 核心技术架构

### 一、 OmniNPC 核心系统分层架构

整个系统可以看作是一个“三明治”结构，从外到内分为四个层级：

- **1. 接入与路由层 (API Gateway / Endpoint)**
  - **负责：** 接收游戏前端（或测试界面）传来的玩家输入文本、当前环境信息，并负责将最终结果打包返回。
- **2. 认知流转层 (Cognitive Flow / Agent Core) —— 系统的“大脑”**
  - **负责：** 核心逻辑的执行。包含输入解析、触发检索、生成内心独白（Inner Monologue）、生成对外台词，以及决定是否调用外部工具。
- **3. 记忆与知识引擎层 (Memory & Graph Engine) —— 系统的“海马体”**
  - **负责：** 数据存储与检索。由轻量级向量数据库（存文本块）和图数据库（存实体关系）双轨并行。
- **4. 离线沙盒演化层 (Offline Sandbox Tick) —— 系统的“潜意识”**
  - **负责：** 在后台默默运行的定时任务（Tick）。处理记忆的自动衰减（遗忘曲线）以及 NPC 之间的私下社交演化。

### 二、 核心数据流向图 (Data Flow)

想象一下，玩家在游戏中对傲娇师姐（NPC）输入了一句话：“师姐，我刚才把师傅最心爱的花瓶打碎了，救救我！”

以下是这句话在 OmniNPC 引擎中经历的完整**数据流转生命周期**：

Plaintext

```
[游戏前端] 玩家输入: "师姐，我把花瓶打碎了..." 
   │
   ▼
【1. 接入层】组装请求 (Payload = 玩家输入 + 当前地点 + 师姐当前好感度)
   │
   ▼
【2. 记忆检索机制】(Perception)
   ├──▶ 查询向量数据库 (ChromaDB)：检索最近5轮的对话 (工作记忆) + 过去是否有“花瓶”相关的事件 (情景记忆)。
   └──▶ 查询图数据库 (GraphRAG)：检索 `[玩家]-[师傅]`、`[师姐]-[师傅]` 的关系网权重 (语义记忆)。
   │
   ▼
【3. 组装终极 Prompt】(Context Assembly)
   将检索到的记忆 + 师姐的系统人设 Prompt + 玩家的输入，拼接成一个巨大的上下文发送给大模型 (LLM)。
   │
   ▼
【4. 内心独白生成】(Inner Monologue - 第一次 LLM 调用或按格式输出)
   LLM 推理思考 (仅后台可见): 
   "师姐内心OS: 这笨蛋怎么又闯祸了！师傅出关肯定要大发雷霆。虽然我很想骂他，但按我的性格，绝不会看着他受罚。先吓唬吓唬他，再帮他想办法。"
   │
   ▼
【5. 决策与行动生成】(Action & Generation)
   结合内心独白，LLM 结构化输出两个结果：
   ├──▶ [文本台词]: "哼，你还有脸来找我？去后山玄冰洞躲着，我不叫你别出来！"
   └──▶ [工具调用 (Tool Use)]: {"action": "update_state", "target": "player_location", "value": "玄冰洞"} 
   │
   ▼
【6. 状态回调与更新】(State Callback)
   引擎解析 Tool Use 的 JSON，通过 API 告诉游戏前端：“把玩家的坐标移到后山”。
   │
   ▼
【7. 记忆沉淀与图谱刷新】(Memory Consolidation)
   提取本次交互的核心实体（玩家、师傅、花瓶、闯祸）。
   ├──▶ 将原始对话向量化，存入向量数据库。
   └──▶ 刷新图数据库：增加一条新关系 `[玩家] --(打碎花瓶, 欠人情)--> [师姐]`。
   │
   ▼
[返回前端] 渲染师姐的台词，并触发场景切换动画。
```

### 三、 离线演化流（后台数据流向）

这是不依赖玩家输入的独立循环，通常每隔一段时间（比如现实时间 1 小时，或游戏内 1 天）触发一次：

1. **记忆衰减 (Memory Decay):** 定时任务遍历向量数据库，运用艾宾浩斯公式降低旧记忆的权重。权重低于阈值的日常废话被物理删除，高权重的核心事件被固化。
2. **流言传播 (Rumor Spreading):** 抽取两个未在玩家视线内的 NPC（例如师傅和师姐）。后台 LLM 根据图数据库的关系网络，推演他们可能会聊什么。
3. **图谱暗改 (Graph Mutation):** 师傅从师姐那里“隐约”得知了花瓶碎了，图数据库中 `[师傅]` 对 `[玩家]` 的好感度节点在后台悄悄下降。玩家下次遇到师傅时，会毫无防备地直接迎来一场“狂风暴雨”。
